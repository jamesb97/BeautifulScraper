# What is the TDD Workflow Process known as?

The TDD workflow process is commonly known as **Red-Green-Refactor**.

Here's what each phase means:

1. **Red** — Write a failing test first. The test describes the behavior you want, but since the code doesn't exist yet, it fails.

2. **Green** — Write the minimum amount of code necessary to make the test pass. Don't worry about elegance yet — just make it work.

3. **Refactor** — Clean up the code while keeping the tests green. Improve structure, remove duplication, and enhance readability without changing behavior.

This cycle repeats continuously, driving the design of the codebase through tests rather than writing tests after the fact.
