# Broad Network Access

**Broad Network Access** is one of the **five essential characteristics of cloud computing** as defined by NIST (National Institute of Standards and Technology).

It means that cloud services and resources are available over the network and can be accessed through **standard mechanisms** (like HTTPS) using a wide variety of client devices and platforms — without needing specialized or proprietary hardware/software.

In simple terms: **you can reach the cloud from anywhere, on anything.**

Examples of "broad" access in practice:

- Accessing your AWS S3 bucket from a laptop, phone, or tablet
- Using a REST API from a Java backend, Python script, or browser
- Connecting to a cloud database from different operating systems
- Consuming a SaaS app (like Gmail or Salesforce) from any modern browser

---

The full NIST list of five essential cloud characteristics for reference:

| #   | Characteristic             | What it means                                              |
| --- | -------------------------- | ---------------------------------------------------------- |
| 1   | **On-demand self-service** | Provision resources without human interaction              |
| 2   | **Broad Network Access**   | Available over the network via standard protocols          |
| 3   | **Resource Pooling**       | Resources shared across multiple consumers (multi-tenancy) |
| 4   | **Rapid Elasticity**       | Scale up/down quickly based on demand                      |
| 5   | **Measured Service**       | Usage is monitored, controlled, and billed (pay-as-you-go) |

---

This concept is foundational to cloud — the whole value proposition of cloud depends on services being broadly accessible rather than locked to a specific location or device.
