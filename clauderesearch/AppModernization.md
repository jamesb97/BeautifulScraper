# Stages of App Modernization Regarding Application Development

App modernization efforts take effect across **all stages** of application development, but conventionally it is said to apply to the **entire application lifecycle** — from the legacy/existing state through to the modernized cloud native state.

More specifically, app modernization takes effect at these stages:

| Stage                   | How Modernization Applies                                                      |
| ----------------------- | ------------------------------------------------------------------------------ |
| **Design/Architecture** | Rethinking monolithic design into microservices or cloud native architecture   |
| **Development**         | Adopting modern languages, frameworks, APIs, and dev practices (CI/CD, DevOps) |
| **Deployment**          | Containerizing apps (Docker), orchestrating with Kubernetes, moving to cloud   |
| **Operations/Runtime**  | Implementing observability, auto-scaling, resilience, and service mesh         |
| **Maintenance**         | Retiring technical debt, refactoring legacy code, updating dependencies        |

The modernization effort is often described through the **"6 R's" framework**:

- **Rehost** — Lift and shift to cloud (minimal changes)
- **Replatform** — Lift and optimize (some cloud benefits)
- **Refactor/Re-architect** — Redesign for cloud native
- **Repurchase** — Switch to a SaaS solution
- **Retire** — Decommission unneeded apps
- **Retain** — Keep as-is for now

The core idea is that modernization is **not just a development concern** — it touches every stage from how the app is designed all the way through how it is operated and maintained in production.
